## 4. SL 초과 원인 분석

### 4.1 vectorbt SL/TP 체결 규칙 (테스트 검증)

| 규칙 | 동작 |
|---|---|
| 진입가 | entry 봉의 **close** |
| SL 트리거 | 다음 봉부터 `low <= SL가`이면 발동 |
| SL 체결가 (정상) | open >= SL가 → **정확히 SL가에 체결** |
| SL 체결가 (갭하락) | open < SL가 → **open에 체결** (SL 초과 손실) |
| SL+TP 동시 도달 | **SL 우선 체결** (비관적 가정) |

### 4.2 SL 초과 현황

- 373건 중 **49건(13%)이 SL 2% 초과** (slippage+수수료 감안)
- 갭하락(open < SL가): 5건(10%) — 나머지 44건은 slippage+수수료 누적
- 최악 초과폭: 2.81%p (-4.81% 거래)

### 4.3 PnL 영향

| 항목 | 값 |
|---|---|
| SL 초과 추가 손실 | $15,553 |
| 전체 순수익 | $103,450 |
| 추가 손실 비중 | **15.0%** |

### 4.4 실거래 executor 현황 및 개선

**변경 전**: SL을 `triggerPrice`만 지정 → stop market order → 슬리피지 미제어

**변경 후**: SL `triggerPrice`에 0.1% 버퍼 추가
- long 포지션: SL가보다 0.1% 위에서 트리거 → 체결가가 SL가에 근접
- short 포지션: SL가보다 0.1% 아래에서 트리거

Stop limit 전환은 미체결 리스크가 방어형 전략에 더 위험하므로 채택하지 않았다.

### 5.2 엣지 분해

- 수수료 0.5x 증가당 수익률 **~1.1%p 선형 감소** — 급격한 붕괴 없음
- **수익률 = 0 되는 수수료 배수: 4.2x** (수수료 0.23%까지 견딤)
- 현재 Bybit taker 0.055% 대비 **4배 이상 여유**

### 5.3 거래당 비용 비중

| 시나리오 | 평균 PnL/거래 | 평균 수수료/거래 | 비용 비중 |

### 5.4 결론

**엣지는 견고하다.** 수수료 2배에서도 수익 양수(+4.76%), 샤프 1.49를 유지한다. 슬리피지가 수수료보다 큰 위협이며, Bybit maker 수수료(0.02%) 활용 시 비용을 추가 절감할 수 있다.
